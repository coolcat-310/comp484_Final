Overview:

Django is a Python web development framework, aimed at rapid development and deployment.

One of the more common questions people have is "which framework" they should use. There are quite a few for Python, with Django and Flask being the two most popular.

Flask is more of what we call a "micro" web framework. It is much "lower level" than Django is. This allows for more customization and control for the developer.

Django is much more of a higher-level framework, and imposes a set structure on the developer.

Django has many features the makes it extremely useful and popular framework

Fast but not bloated – it does a great job cutting the time it takes to make a website

Security is a priority – provides tools for such, it provides user authentication system, sql injection cross site scripting, etc.

Administration Portal – it takes so much time to create admin pages that interact with your back end, Django shifts with it

Completely Scalable – can continue to add hardware and it will take advantage of it, so if your site gets big and you need to find more hardware you don’t need to worry about needing to port your entire code base to different frame work

Successfully used on large projects:

Installation: (windows)

Python:

Django requires python – get the latest version at https://www.python.org/download/ . Download the executable installer and run it. Check the box next to AddPython 3.5 to PATH and then click Install Now.

- After installing python you can check which version you downloaded by opening command prompt and executing:

Python --version

PIP:

pip is a package manage for python. Pip is use to install python packages from the command line. If you installed Python 2 >=2.7.9 or Python 3 >=3.4 binaries pip is already installed.

Django:

Using a virtual environment Django can be installed easily using pip. In the command prompt making sure the virtual environment is active execute:

pip install Django

this will install the latest version of Django

If you want to check which version you can execute in the command prompt:

django-admin –version

Install Vritualenv:

The goal of virtualenv is to prevent different versions of libraries/packages from messing with each other.

Think of virtualenv as a completely isolated container within your computer, where you can utilize any version of Python and install libraries/packages and it won’t affect anything outside that container. It’s like an isolated, soundproof room.

Install virtualenv with the following command:

$ pip install virtualenv

Django Install:

Set up development structure:

$ mkdir django15_project

$ cd django15_project

$ virtualenv env

$ source env/bin/activate

To Exit virtualenv type:

$ deactivate

Now install Django:

$ pip install django==1.5.12

to check for installed version:

$ python

>>> django.get_version()

‘1.5.12’

>>>

Project Setup:

$ django-admin.py startproject my_django15_project

This creates a new directory called “my_django15_project” with the basic Django directory and structures
